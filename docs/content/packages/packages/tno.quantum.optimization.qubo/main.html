optimization.qubo

1.0.1

This package provides a comprehensive suite of tools for creating QUBO objects, defining QUBO solvers, and using them to solve QUBO problems. It also includes features for pre- and post-processing, as well as the ability to create pipelines that integrate all these components seamlessly.

The qubo package can be installed using pip:

pip install tno.quantum.optimization.qubo

By default, the package is installed without external solver dependencies. You can specify which QUBO solvers you would like to install. Available options are [dwave, qubovert, dqo]. Alternatively, you can install all solvers simultaneously using the '[all]' option:

pip install tno.quantum.optimization.qubo'[all]'

Example

The following example shows how to construct a QUBO object.

>>> from tno.quantum.optimization.qubo import QUBO
>>> qubo = QUBO([
...     [ 1, -2,  3],
...     [-4,  5, -6],
...     [ 7, -8,  9]
... ])

One can use the SolverConfig class to find and instantiate available solvers to solve this QUBO, as shown in the following example.

>>> from tno.quantum.optimization.qubo import SolverConfig
>>> list(SolverConfig.supported_items())
['bf_solver',
'custom_solver',
'daqo_solver',
'd_wave_clique_embedded_simulated_annealing_solver',
'd_wave_clique_sampler_solver',
'd_wave_embedded_simulated_annealing_solver',
'd_wave_parallel_embedding_solver',
'd_wave_sampler_solver',
'd_wave_tiling_solver',
'digital_adiabatic_quantum_optimization_solver',
'exact_sampler_solver',
'kerberos_sampler_solver',
'leap_hybrid_solver',
'neighborhood_solver',
'pipeline_solver',
'qaoa_solver',
'rs_solver',
'random_sampler_solver',
'sa2_solver',
'simulated_annealing_solver',
'steepest_descent_solver',
'tabu_solver',
'tree_decomposition_solver']
>>> solver = SolverConfig(name='bf_solver').get_instance()
>>>
>>> # Solve the QUBO
>>> result = solver.solve(qubo)
>>> result.best_bitvector
BitVector(000)

Similarly, one can use the PreprocessorConfig and PostprocessorConfig classes to find and instantiate available preprocessors and postprocessors.

compute_bounds(**kwargs)[source]

Compute an upper and lower bound for the QUBO problem.

First the objective function is convexified and then the integrality constraints are dropped (see https://link.springer.com/article/10.1007/s10107-005-0637-9). The resulting (convex, constrained) continuous optimization problem is solved via TNC algorithm. The solution gives a lower bound. An upper bound is obtained by rounding the solution to the relaxation.

Parameters:

kwargs (Any) – Optional keyword arguments to pass to TNC algorithm.

Return type:

ndarray[tuple[Any, ...], dtype[float64]]

Returns:

Solution to the convex relaxation of the QUBO.

classmethod from_bqm(bqm, *, sort_labels=False)[source]

Construct a QUBO instance from BinaryQuadraticModel.

This method converts a given BinaryQuadraticModel into a QUBO representation. If the BinaryQuadraticModel variable type is SPIN, it will be automatically converted to BINARY with a warning. The resulting QUBO matrix is constructed using the linear and quadratic coefficients from the BinaryQuadraticModel, and returned along with the list of variable labels used in the conversion. Optionally, the labels can be sorted as well.

Parameters:

  • bqm (BinaryQuadraticModel) – The binary quadratic model to convert.

  • sort_labels (bool) – If True, variable labels will be sorted before constructing the QUBO matrix. Defaults to False.

Return type:

tuple[Self, list[Any]]

Returns:

Tuple containing a QUBO instance representing the input BinaryQuadraticModel and a (optionally sorted) list of variable labels corresponding to the order used in the QUBO matrix.

Example

The example below shows how to generate a QUBO object from a BinaryQuadraticModel.

>>> from dimod import BinaryQuadraticModel
>>> bqm = BinaryQuadraticModel({"x": 1, "y": 2, "z": 3}, {("x", "y"): 4}, 5, "BINARY")
>>> qubo, labels = QUBO.from_bqm(bqm)
>>> labels
['x', 'y', 'z']
>>> qubo.matrix
array([[1., 0., 0.],
       [4., 2., 0.],
       [0., 0., 3.]])
>>> qubo.offset
5.0

spectral_gap(A, B, *, plot=True)[source]

Create a spectral gap figure and compute the minimum spectral gap.

Simulates a quantum annealing process as described by the Hamiltonian

\[H_\text{ising} = - \frac{A(s)}{2} \left(\sum_i {\sigma_x^{(i)}}\right) + \frac{B(s)}{2} \left(\sum_i h_i {\sigma_z^{(i)}} + \sum_{i > j} J_{i, j} {\sigma_z^{(i)}} {\sigma_z^{(j)}}\right)\]
where

  • \(s \in [0, 1]\) is the time parameter,

  • \(A(s)\) and \(B(s)\) are the annealing functions,

  • \(\sigma_x^{(i)}\) and \(\sigma_z^{(i)}\) are Pauli matrices acting on qubit \(i\),

  • \(h_i\) are the qubit biases,

  • \(J_{i, j}\) are the coupling strengths.

The spectral graph figure is a plot of the energy eigenvalues of \(H_\text{ising}\) over time. The minimum spectral gap is the minimum difference between the two lowest energy eigenvalues.

Not recommended for use when size of QUBO \(\ge 10\).

Parameters:

  • A (list[float]) – List of values of \(A\) for the annealing process.

  • B (list[float]) – List of values of \(B\) for the annealing process.

  • plot (bool) – Boolean value, if True (default) we plot here. If False we do not plot here.

Return type:

tuple[float, Figure]

Returns:

The minimum spectral gap and a matplotlib figure containing the spectral gap figure.

Raises:

ValueError – If A and B do not have equal length, or have length zero.
